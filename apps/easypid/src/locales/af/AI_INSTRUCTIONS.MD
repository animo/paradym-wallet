## Afrikaans Translation Rules for Identity Wallet App

### **Tone & Style**
- Use informal, friendly tone throughout (Afrikaans naturally uses casual register)
- Keep translations natural and conversational, not literal
- Match the directness and brevity of the original English
- Be clear and straightforward - avoid unnecessary complexity

### **Key Terminology**
- **Wallet** = "Beursie" or "Portefeuille" (use "beursie" for app context - more familiar)
- **Card** = "Kaart" (for digital credentials)
- **Credential** = "Bewys" or "Akkreditasie"
- **PIN** = "PIN" (keep as-is)
- **QR code** = "KR-kode" (or "KR-kode" with Afrikaans naming)
- **Biometrics** = "Biometrie" or "Biometriese verifikasie"
- **eID card** = "eID-kaart"
- **Organization** = "Organisasie"
- **Verifier** = "Verifieerder"
- **Issuer** = "Uitgewer"
- **Scan** = "Skandeer"
- **Share/Sharing** = "Deel/Deling"
- **Authentication** = "Verifikasie"
- **Verification** = "Verifikasie"

### **Common Phrases**
- "Go to" → "Gaan na" or "Na"
- "Set up" → "Stel in"
- "Please try again" → "Probeer asseblief weer"
- "Something went wrong" → "Iets het verkeerd gegaan"
- "Continue" → "Gaan voort"
- "Cancel" → "Kanselleer"
- "Stop" → "Stop"
- "Back" → "Terug"
- "Add" → "Voeg by"
- "Accept" → "Aanvaar"
- "Decline/Reject" → "Weier"

### **Placeholder Handling**
- Preserve all placeholders exactly as written: `{0}`, `{name}`, `{userName}`, etc.
- Adjust Afrikaans sentence structure around placeholders naturally
- Maintain proper Afrikaans grammar with placeholders

### **Specific Guidelines**
- **Time expressions**: "Today" = "Vandag", "Yesterday" = "Gister", "Tomorrow" = "Môre"
- **Confirmation dialogs**: Use "Is jy seker dat jy..." for "Are you sure..."
- **Permission requests**: Use "Laat toe" for "Allow/Enable"
- **Settings/Setup**: "Stel in" for "Set up", "Instellings" for "Settings"
- **Error messages**: Start with what happened, then suggest action - be helpful
- **Success messages**: Use positive tone (e.g., "Suksesvol bygevoeg!")

### **Grammar Notes**
- Use simple verb forms (Afrikaans has no complex conjugations)
- Avoid overly formal constructions like "u" - stick to "jy/jou"
- Use straightforward word order (Subject-Verb-Object)
- For compound phrases, prioritize clarity and natural Afrikaans feel

### **What NOT to Translate**
- Brand names (keep as-is)
- Technical identifiers in metadata
- File extensions
- URLs or technical paths
- Acronyms like "PIN", "QR", "AI", "eID"

### **Context Awareness**
- Consider the user's emotional state (error vs success vs neutral)
- Match urgency level of original message
- Security-related messages should be clear but not alarming
- Keep messages concise - Afrikaans can be more compact than English
